# Data Model: 记忆注入策略升级

**Feature**: `004-memory-lazy-inject`
**Status**: No schema changes

---

## Schema (unchanged)

This feature introduces zero changes to the `memory.db` schema. The `entries` table defined in SPEC 002 remains:

```sql
CREATE TABLE entries (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  namespace TEXT NOT NULL,
  key TEXT NOT NULL,
  value TEXT NOT NULL,
  source_session_id TEXT,
  version INTEGER NOT NULL DEFAULT 1,
  expired INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now')),
  project_id TEXT NOT NULL,
  UNIQUE(project_id, namespace, key)
);
```

## View Model: Injection Index

The injection index is a **runtime-only view** generated by `buildInjection()`. It is never persisted.

| Field | Source | Description |
|-------|--------|-------------|
| `namespace` | `entries.namespace` | Grouping key for entries |
| `keys` | `entries.key` (aggregated) | Comma-separated key names, filtered `expired = 0` |
| `count` | `COUNT(entries.key)` | Entry count per namespace (used for collapse logic) |
| `suggestion` | Static mapping | "必读" for meta/decisions, "按需" for risks/learnings |

### Index Generation Logic

```
total_entries = 0
FOR ns IN namespaces:
  keys = SELECT key FROM entries WHERE namespace = ns AND expired = 0
  ns_entries[ns] = { keys, count: len(keys) }
  total_entries += len(keys)

IF total_entries == 0:
  RETURN "当前无共享记忆条目"

IF total_entries > 20:
  // Collapsed mode
  FOR ns IN namespaces WHERE ns_entries[ns].count > 0:
    OUTPUT "| {ns} | ({count} 条) | {suggestion} |"
ELSE:
  // Full index mode
  FOR ns IN namespaces WHERE ns_entries[ns].count > 0:
    OUTPUT "| {ns} | {keys.join(', ')} | {suggestion} |"
```

### Suggestion Mapping

```
project/meta      → "必读"
project/decisions → "必读"
project/risks     → "按需"
project/learnings → "按需"
```
