# Guide: 多文档交叉对照

> 加载条件：Phase 1 文档侦察时 Read；当输入包含 SPEC/PRD/PLAN/TASK/设计文档中的任意两类时必须 Read。

## §1 目标

把多份文档转换为可审计的一致性报告，给 Contract 生成提供事实基线。报告必须保留已覆盖、遗漏、矛盾点，不能为了让 Contract 更顺滑而隐藏不一致。

## §2 文档分类

| 文档类型 | 识别线索 | 主要用途 |
|----------|----------|----------|
| SPEC | `spec.md`、接口定义、Acceptance Criteria、需求编号 | AC、接口契约、范围边界 |
| PRD | 用户故事、验收条件、业务目标 | AC、目标陈述、方向基线 |
| PLAN | 技术方案、实施步骤、架构取舍 | 实现覆盖、复杂度因素、依赖 |
| TASK | 任务列表、checkbox、分工、里程碑 | 执行粒度、遗漏步骤、并行性 |
| 设计文档 | ADR、design、architecture、docs/superpowers/specs | 决策约束、协议层/DB schema 变化 |

## §3 AC 提取优先级

按以下顺序提取 acceptance criteria；高优先级存在时不得被低优先级覆盖：

优先级：`SPEC AC章节 > PRD 验收条件 > PLAN 成功标准`

1. `SPEC` 的 AC / Acceptance Criteria / 验收标准章节
2. `PRD` 的验收条件、用户可见成功条件
3. `PLAN` 的成功标准、完成标准、验证标准

### 门控

| 检查项 | 通过条件 |
|--------|----------|
| AC 可验证 | 每条以返回、通过、包含、不抛出、生成、拒绝、保持等可验证动词开头 |
| AC 可追溯 | 每条能指向源文档路径和章节 |
| AC 不降级 | PLAN/TASK 未覆盖时仍保留 SPEC/PRD 的 AC，并标记遗漏 |

## §4 矛盾检测

按三条主线对照：

1. `SPEC 接口定义` vs `PLAN 实现步骤`
   - 参数名、字段、返回结构、错误模式不一致时标记矛盾。
   - SPEC 要求的 endpoint/tool/CLI 行为在 PLAN 中改名或合并时标记矛盾。
2. `PLAN 实现步骤` vs `TASK 粒度`
   - PLAN 需要多阶段实现，但 TASK 只给一个笼统任务时标记粒度不足。
   - TASK 包含 PLAN 未提及的额外工作时标记范围漂移。
3. `SPEC 范围边界` vs `TASK 执行项`
   - TASK 缺少 SPEC 定义的接口、校验、错误路径、文档更新时标记遗漏。
   - TASK 增加 SPEC 明确不做的功能时标记矛盾。

### 矛盾等级

| 等级 | 定义 | Contract 处理 |
|------|------|---------------|
| Critical | 会改变 AC、接口契约或交付范围 | 停止并要求 PM 裁决 |
| Warning | 不阻塞生成，但影响复杂度或风险 | 写入 `complexity.factors` |
| Note | 命名、章节或措辞轻微差异 | 写入一致性报告 |

## §5 遗漏标记

必须标记以下遗漏：

| 遗漏类型 | 判定方式 | 记录位置 |
|----------|----------|----------|
| SPEC 定义但 PLAN 未覆盖的接口 | SPEC 有接口/字段/错误模式，PLAN 无实现步骤 | 遗漏列表 + complexity.factors |
| PLAN 需要但 TASK 缺失的步骤 | PLAN 有实施动作，TASK 无对应 checkbox/步骤 | 遗漏列表 |
| AC 无验证任务 | AC 存在，但 TASK/PLAN 没有测试、审计或验收动作 | 遗漏列表 + pm_duties |
| 协议层/DB schema 变更未计划迁移 | 文档提到协议或 schema，但 PLAN/TASK 无兼容/迁移步骤 | 矛盾点或遗漏列表 |

## §6 输出格式

使用三列表一致性报告：

```markdown
## 一致性报告

### 已覆盖
| # | 来源 | 覆盖文档 | 说明 |
|---|------|----------|------|
| 1 | <SPEC §x> | <PLAN §y / TASK Tz> | <接口/AC 已覆盖> |

### 遗漏
| # | 来源 | 缺失位置 | 影响 |
|---|------|----------|------|
| 1 | <SPEC §x> | <PLAN/TASK> | <缺少实现/验证步骤> |

### 矛盾点
| # | 来源 A | 来源 B | 等级 | 说明 |
|---|--------|--------|:---:|------|
| 1 | <SPEC §x> | <PLAN §y> | Critical/Warning/Note | <不一致内容> |
```

## §7 禁止项

| 禁止行为 | 原因 |
|----------|------|
| PLAN 未覆盖就删除 SPEC AC | 自行降级 |
| 把矛盾点改写成模糊风险 | 降低 PM 裁决可见性 |
| 只输出总结不列三列表 | 无法审计 |
| 用 TASK 反向覆盖 SPEC 接口 | 低优先级覆盖高优先级 |
